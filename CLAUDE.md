# 執筆ルール

このドキュメントは、記事を執筆する際の文体・構成・表現のルールをまとめたものです。

## 記事構成の基本パターン

### 1. 見出し構造
- **3段階の階層構造**: `#` (H1) → `##` (H2) → `###` (H3)
- H1は記事タイトルのみ
- H2で大きなセクションを区切る
- H3で詳細項目を列挙する

### 2. 記事の流れ
記事は以下の順序で構成する：

1. **導入部（定義）**: 「〜とは」で主題の定義を示す
2. **基本要素**: 核となる要素や概念を番号付きリストで列挙
3. **種類・分類**: 具体的なパターンや種類を説明
4. **コツ・テクニック**: 実践的なヒントを提供
5. **問題と解決策**: よくある課題とその対応方法
6. **まとめ**: 全体を振り返り、実践を促す

## 文体ルール

### 基本トーン
- **「です・ます」調**: 丁寧で親しみやすい文体を使用
- **簡潔さ**: 1文は短く、要点を明確に
- **肯定的**: 前向きで実践的なトーンを保つ

### 説明の仕方
- **定義は明確に**: 「〜とは、...のことです」の形式
- **因果関係を明示**: 「〜することで」「〜によって」を使用
- **行動を促す**: 「〜してください」で具体的な指示を出す
- **専門用語には説明を**: 初出の専門用語は必ず説明を添える

## 表現・フォーマットルール

### 1. リストの使用
**箇条書き（ハイフン）の使用場面:**
- 順序が重要でない項目の列挙
- 詳細説明や補足事項
- 要素の特徴や性質の説明

**番号付きリストの使用場面:**
- 手順やステップの説明
- 優先順位がある項目
- カウント可能な要素の列挙

### 2. 強調表現
- **太字（`**`）**: 重要なキーワードや要点を強調
  - リストの見出し部分に使用（例：「**具体的に書く**: 説明文」）
  - 重要な概念や用語の強調

### 3. コードブロックの使用
- 具体例を示す際に使用
- コマンド、コード、プロンプト例などを記載
- バッククォート3つ（` ``` `）で囲む

### 4. 表の使用
- 対比構造を示す際に使用
- 問題と解決策の対応を示す
- カテゴリと説明の対応を整理

## 避けるべき表現

### ❌ 使用しない表現
- 曖昧な表現: 「おそらく」「たぶん」「かもしれません」
- 過度に複雑な文章: 1文に複数の主張を詰め込まない
- 否定的なトーン: 「〜できません」ではなく、代替案を提示
- 専門用語の乱用: 説明なしで専門用語を連発しない
- 口語的すぎる表現: 「めっちゃ」「すごく」などのカジュアルな表現

### ❌ 避けるべき構成
- 見出しなしの長文
- 箇条書きや表を使わない密集した文章
- 具体例のない抽象的な説明のみ
- まとめのない唐突な終わり方

## セクション別の執筆ガイド

### 導入部（「〜とは」セクション）
```markdown
## [主題]とは

[主題]とは、[簡潔な定義]です。[効果や意義を1〜2文で説明]。
```

### 基本要素セクション
```markdown
## 基本的な[主題]の要素

### 1. [要素名]
- [詳細1]
- [詳細2]
- [詳細3]

### 2. [要素名]
- [詳細1]
- [詳細2]
```

### コツ・テクニックセクション
```markdown
## 効果的な[主題]のコツ

1. **[要点]**: [説明文]
2. **[要点]**: [説明文]
3. **[要点]**: [説明文]
```

### 問題解決セクション
```markdown
## よくある問題と解決策

| 問題 | 解決策 |
|------|--------|
| [問題1] | [解決策1] |
| [問題2] | [解決策2] |
```

### まとめセクション
```markdown
## まとめ

[主題]は[重要性や本質]です。練習を重ねることで、[期待される成果]。
```

## 実践チェックリスト

記事執筆後、以下を確認する：

- [ ] H1, H2, H3の階層構造が適切か
- [ ] 「です・ます」調で統一されているか
- [ ] 箇条書きや表を効果的に使用しているか
- [ ] 具体例（コードブロック等）が含まれているか
- [ ] 重要な用語に太字で強調がされているか
- [ ] 専門用語に説明が添えられているか
- [ ] まとめセクションがあるか
- [ ] 読者に行動を促す表現があるか
- [ ] 曖昧な表現や否定的なトーンがないか
- [ ] 1文が簡潔で読みやすいか
